This page is for people who work in local authorities. It outlines a common pattern used for delivering resident's parking permits services effectively. Read more about service patterns.

What is the user need?

Residents need a place to park their car near their home. Parking permit services make sure that parking spaces are only occupied by residents by making it illegal to park in a given area without a permit.

We recommend that local authorities provide the parking permit service using a user centric approach.

What do councils need from users to deliver the service?

Depending on local authority needs, the user may be asked to provide a combination of:

  • proof that the user lives in the area
  • the registration number of the vehicle to be permitted
  • payment (this is to support the cost of the service and will vary across councils and potentially within service, depending on the type of vehicle, engine size, parking zone and others)

What is the user journey?

At a high level, these are the steps users should need to go through.

  1. Awareness

    Users need to find out about the parking permits service before they need to park their car. There are a number of ways users could find out about the service, for example:

    • information from the council when they sign up for council tax
    • information from their estate agent when they buy their house
    • signage in the street

    Local authorities should make sure the correct information is being provided through all channels within their control.

  2. Finding the right council

    Once the user has found out they need a permit, they need to get to the right council's website to apply. Users might start by typing 'parking permit' into a search engine, or they might start on the council's website where they think they need a permit.

    The parking permits page on GOV.UK should send them to the right council's website once they enter their postcode. Councils should make sure the postcodes that they serve are linked to the right page on their website and email any corrections to local-direct-admin@digital.cabinet-office.gov.uk.

    Users may also go straight to the website for the council that they think they need to get a permit from. This makes it very important that councils clearly state which area their parking permit service covers as the user may be in the wrong place. Read the next section for details on this.

  3. Service start page

    Now the user is in the right place, the page on the council's website should communicate:

    • the different types of permit the user can apply for (if applicable)
    • a postcode look up tool to check eligibility for available permits based on selected address (if deemed necessary by the council due to the complexity of available permits)
    • permit eligibility requirements
    • where the permit they are applying for will allow them to park
    • at what time the permit they are applying for will allow them to park
    • how much the permit will cost
    • how long the permit is valid for
    • what information they need about the car for example number plate (registration number) of the vehicle
    • how long the process usually takes
    • what to do if they need assistance with the application
    • if the service differs from this pattern, include any other information the user needs to provide

  5. Confirm address and eligibility

    GOV.UK Verify will send the council the user's address, as known by the user's chosen identity provider. On returning from the GOV.UK Verify process, the council should ask the user to confirm whether or not this is their current address.

    At this point the council should check whether or not the person or the household already has a parking permit. For some councils this may affect the price, for others it might mean the user can not have another permit.

    See the example page asking users to confirm their name and address

    If the user says the address returned is not correct, they are likely to be a new resident whose address has not been updated with their identity provider. The user should then be asked to enter their current address and warned that they must not enter false information.

    See the example page asking users to enter their current address

    If the permit is physical, councils will get extra confidence in the address by mailing the permit to the address given. Councils should mention they'll be using the address for this purpose on the confirmation page.

    If the permit is digital and councils want extra confidence of eligibility, they can issue a temporary permit for a shorter time period, for example 3 months. After this time, the user's identity provider should have the new address and the user can reapply.

    1. Does the user live in the area?

      At this point the council should check whether or the user lives in the eligible area, and then show a message to the user confirming they are eligible. This is done by checking the user's address against a council held database of parking permit zones.

      See the example page which says the user is not eligible because of their address

    2. Does the user or the user's address already have a permit?

      Some councils have limits on the number of people who can get a permit, or number of permits allowed per address. At the same time as check addresss eligibility (from the point of view of the user), the council should check whether or not there are any permits currently assigned to the person or their address and issue an ineligible message if applicable.

      See the example page which says the user is not eligible because they have exceeded the number of permits allowed

    3. Is the user eligible for multiple permits?

      If the user is elibible for multiple permits, for example visitors permits, the council should ask which type of permit the user would like to apply for.

      See the example page asks the user to select which permit they wish to apply for.

  7. Enter vehicle details

    The council should now ask the user to enter the registration number of the car they wish to pay a permit for.

    If the price of the permit varies depending on the vehicle type, emissions, make or any other condition, the council should present information or a tool that allows correct calculation of the permit price

    See the example page of asking for a registration number

  8. Pay

    The fee will vary based on the fee calculation rules determined by each individual council. In some cases the permit can be issued free of charge, for example, if the resident is the first member of the household to apply.

    See the example page of accepting payments.

    Councils can accept payment via a number of channels, including checks, bank transfers or online payments. Councils can find a supplier on the Digital Marketplace.

    In the future, councils will be able to accept payments using GOV.UK Pay.

  9. Receive permit confirmation

    After payment, the user needs to receive a confirmation of payment and an acknowledgement that their application has been successful. They should also receive information about how the permit will be enforced (See step 8). If the payment cannot be accepted or if the application fails for some reason (i.e. times out) the user should be shown an on-screen message that specifies the next steps a user should take to continue with the application.

    The service should ask the user how they would like to receive this confirmation of payment, offering the choice of email, letter or text. See the example page on how to do this.

    In the future, councils will be able to send this confirmation using GOV.UK Notify.

  10. Contacting users

    You may want to capture users contact details so they can be contacted if things go wrong with their application, or for use in future for renewal reminders.

    You should allow users to select their prefered contact method and only contact them in this way in future.

    See the example page asking users to provide their prefered contact details.

  11. Checking permits

    It's better for users and cheaper for government if the permits are issued electronically. Wardens check the permits by entering number plates into a database.

    Councils who haven't yet moved over to an electronic permit system, should have a plan in place to move over.

    In the meantime, councils should mention in the payment receipt

    • how long they will need to wait for a permit to be posted to them
    • if they can pick up their permit from a council office to get it faster and where from
    • if they can print a temporary permit themselves while they wait for a permanent one
    • this information should also be communicated on the council's start page at the beginning of the user journey
    • advice on how customers can park without their permit until they receive it

    This information should also be communicated on the council's start page at the beginning of the user journey.

    Temporary permits should clearly show their expiry date so wardens and users can check them easily.

  12. Renewals

    Councils should remind users when it's time for them to renew their permits. They should do this using the same contact method as used in the payment receipt.

    See the example renewal notification. (page in progress)
