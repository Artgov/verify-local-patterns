User needs identified during the Discovery phase

As a resident applying for a pass, I need:

  • to know how that I’ve done everything correctly, so that I know my application will be successful
  • to know what’s happening with my application, so that I don’t have to call up
  • to find out when I’ll get it
  • to get my bus pass quickly, so that I’m able to travel
  • to know where and how I can use my pass, so I don’t get thrown off a bus

Once applications had been submitted users needed to know that they were being processed, as well as when they would be issued.

These users also tended to be less confident about their applications, and there was a greater need to know that they had completed the application correctly.

What we learnt while testing

At the end of the transaction, users need a clear confirmation that they have completed the task. In addition to this, they also need to understand when and how they will receive their pass.

Several users also wanted to know how and where they would be able to use their pass.

Some users thought the identity verification process was specifically for the bus pass application.

What we changed as a result

Content has been added to this page to explain how the pass can be used.

Content has also been added to mention other potentially relevant services the user can access with their Verify account. This addresses an observed issue where some users thought the identity verification process was specifically for the bus pass application.
